TIMERS of 8051

Timmers are used to produce a delay in the program.
Timers are of two types :
1.HARDWARE - T0,T1
2.SOFTWARE - NOP (nop is written in program)
Two timers T0,T1 are used to produce Two delays.
Each timer is further divided into the following.

graph TB T0 --8 BIT--> TH0; T0 --8 BIT--> TL0; T1 --8 BIT--> TH1; T1 --8 BIT--> TL1;

For every clock pulse the count will be incremented.

The count is calculated bu using the formula :
Count to be loaded = MAX. Count - Desired Count + 1
As soon as the counting is done The Timer overflow flag becomes 1.

  • Here flags are used to store the PENDING status of interrupts.

⭐️ TIMERS :

  • In "TIMERS" the count have to done within the given time.
  • It uses oscillator frequency of 1MHz to count .
    i.e., 12MHz/12 = 1MHz,Hence its frequency cannot be changed.

⭐️ COUNTERS :

  • Time required to complete the given count .
  • In Counters we will connect T0,T1 "EXTERNALLY" , hence its frequency can be changed.

⭐️ TCON :(8-bits)
TCON - Timer control

TF1 TR1 TF0 TR0 IE1 IT1 IE0 IT0
  • First 4 bits belongs to Timer and last 4 bits belong to INTERRUPTS
  • TF1,TF0 - Timer overflow flags
    • these are timer overflow flags for T0,T1.
    • these flags are AUTOCLEARED .
  • TR1,TR0 - Timer run bits
    • TRx = 1 (RUN).
    • TRx = 0 (STOP).
    • these are bit addressable and can be written in the program.
  • IE1,IE0 - Interrupt External
    • As soon as an interrupt is occured their corresponding flags are "SET" (1).
  • IT1,IT0 - Interrupt Triggered
    • these are bit addressable we can select the type of TRIGGER we want in the clk pulse .
    • ITx = 1 (EDGE).
    • Itx = 0 (LEVEL).

⭐️ TMOD : (8-bits)
TMOD - Timer mode

gate c/t M1 M0 gate c/t M1 M0
  • First 4 bits belongs to T1 and last 4 bits belong to T2
  • GATE
    • this bit is used to connect EXTERNAL button to INT0,INT1.
    • When button is connected to INTx,then we cannot send an interrupt.
    • 1 = counting is controlled by ~INTx.
    • 0 = counting is independent of ~INTx.
  • c/t
    • c/t = 1 (counter)
    • c/t = 0 (timer)
  • M1,M0
Modes M1 M0
M0 0 0
M1 0 1
M2 1 0
M3 1 1

1.M0 : 13-bit timer count

graph LR a(CLK from i/p stage ) --8-bit--> b(TLX); b --8-bit--> c(THX); c --> d(TFX); d --> 1(INT);
  • For every clk pulse TL is incremented.
  • Once it reaches its max. limit
  • TH will be incremented.Once it reaches its max. limit.
  • TF is incremented and then sends an interrupt .
  • Max. = 8k byte
  • Created to divide frequency with 32

2.M1 : 16-bit timer/counter

graph LR a(CLK from i/p stage ) --8-bit--> b(TLX); b --8-bit--> c(THX); c --> d(TFX); d --> 1(INT);
  • For every clk pulse TL is incremented.
  • Once it reaches its max. limit
  • TH will be incremented.Once it reaches its max. limit.
  • TF is incremented and then sends an interrupt .
  • Max. = 64k byte
  • Used to produce DELAY

3.M2 : 8-bit timer/counter

  • For every clk pulse TL is incremented.
  • Once it reaches its max. limit .
  • the count given in TLx RELOADS and START counting again with the same i/p given.
  • TF is incremented and then sends an interrupt .
  • It is a continous mode

4.M3 : 2,8-bit Timers using T0

graph LR a(CLK from i/p stage) --8-bit--> b(TL0) ; b --> TF0; 1(fosc/12) --8-bit--> 2(TH0) ; 2 --> TF1;
  • For every clk pulse TL is incremented.
  • Once it reaches its max. limit. It increments TF0.
  • TF0 is incremented and then sends an interrupt .
  • For every clk pulse TH is incremented.
  • Once it reaches its max. limit. It increments TF1.
  • TF1 is incremented and then sends an interrupt .
